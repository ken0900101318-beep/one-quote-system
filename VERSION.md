## 版本命名規則

格式：`vX.Y.Z`

- **X（主版本）**：重大功能更新或架構變更
- **Y（次版本）**：新功能添加
- **Z（修訂版本）**：bug 修復和小改進

範例：
- v1.0.0 → v1.0.1：bug 修復
- v1.0.1 → v1.1.0：新增功能
- v1.9.9 → v2.0.0：重大更新

- 兩個優惠位置：項目後方（單項優惠）+ 總報價單（整單優惠）
- 優惠方式：固定金額 + 折扣比例（兩種都支援）
- 優惠範圍：針對總價優惠（不是單價）
- UI 顯示：業務優惠 -$1,000

**新功能：**

**1. 單項優惠（項目後方）**
- 每個項目有兩個優惠按鈕：
  - 💼 業務優惠（藍色）
  - 🏢 公司優惠（綠色）
- 支援兩種優惠方式：
  - 固定金額（例如：-$5,000）
  - 折扣比例（例如：9折 = 10% off）
- 優惠計算順序：原價 → 業務優惠 → 公司優惠
- 優惠明細顯示：
  - 原價：$70,000
  - 業務優惠：-$5,000（藍色）
  - 公司優惠：-$2,000（綠色）
  - 優惠價：$63,000

**2. 整單優惠（總報價底部）**
- 兩個整單優惠按鈕：
  - 💼 整單業務優惠
  - 🏢 整單公司優惠
- 同樣支援固定金額和折扣比例
- 整單優惠在所有項目優惠之後計算
- 顯示在總計上方：
  - 小計：$500,000
  - 業務優惠：-$10,000 [修改]
  - 公司優惠：-$5,000 [修改]
  - 總計：$485,000

**3. 優惠層級與計算順序**
```
原價（各項目數量 × 單價總和）
↓
單項業務優惠（每個項目可設定）
↓
單項公司優惠（每個項目可設定）
↓
小計
↓
整單業務優惠（針對小計）
↓
整單公司優惠（針對扣除整單業務優惠後的金額）
↓
總計（最終報價）
```

**技術實作：**

**數據結構（單項優惠）：**
```javascript
{
  salesDiscount: 5000,      // 業務優惠金額
  companyDiscount: 2000,    // 公司優惠金額
  // 計算：
  originalTotal = quantity × price
  afterDiscount = originalTotal - salesDiscount - companyDiscount
}
```

**數據結構（整單優惠）：**
```javascript
globalSalesDiscount: 10000,    // 整單業務優惠
globalCompanyDiscount: 5000    // 整單公司優惠
```

**函數清單：**
- setSalesDiscount(index)：設定單項業務優惠
- setCompanyDiscount(index)：設定單項公司優惠
- setGlobalSalesDiscount()：設定整單業務優惠
- setGlobalCompanyDiscount()：設定整單公司優惠
- renderFinalSummary()：更新總計（包含所有優惠）

**UI 優化：**
- 業務優惠：藍色（#3b82f6）
- 公司優惠：綠色（#10b981）
- 贈送：紫色（#8b5cf6）
- 恢復：橘色（#f59e0b）
- 移除：紅色（#ef4444）

**保護機制：**
- 單項優惠不能超過該項目原價
- 公司優惠計算基準：扣除業務優惠後的金額
- 整單優惠不能超過小計
- 整單公司優惠計算基準：扣除整單業務優惠後的金額
