## 🚀 開始前準備

### 1. 清除瀏覽器快取 ⚠️ **必須執行**
```
方法 1（推薦）：
1. 按 F12 開啟開發者工具
2. 右鍵點擊重新整理按鈕
3. 選擇「清除快取並強制重新整理」

方法 2：
使用快捷鍵 Ctrl+Shift+F5 (Windows) 或 Cmd+Shift+R (Mac)

方法 3（最徹底）：
1. 按 Ctrl+Shift+Delete
2. 選擇「快取的圖片和檔案」
3. 時間範圍選「不限時間」
4. 點擊「清除資料」
```

### 2. 開啟 Console 查看訊息
```
按 F12 → 切換到 Console 標籤
```

預期看到的訊息：
```
[分類管理] 頁面初始化
[分類管理] 初始化完成
[分類管理] 渲染分類列表，共 7 項
```

如果沒看到訊息 → **可能還在載入舊版本，再等 5 分鐘重試**

## ❌ 如果遇到問題

### 問題 1：新增分類/項目按鈕無反應
**原因**：可能是舊版本快取  
**解決方法**：
1. 清除快取（Ctrl+Shift+Delete）
2. 等待 5-10 分鐘讓 GitHub Pages 完成部署
3. 使用無痕模式測試
4. 檢查 Console 是否有錯誤

### 問題 2：分類列表空白
**原因**：可能是 API 連線問題  
**解決方法**：
1. 查看 Console 錯誤訊息
2. 查看 Network 標籤，找「getCategories」請求
3. 檢查 API 回應狀態（應該是 200）
4. 確認 localStorage 有 `oneCurrentUser`

### 問題 3：Console 沒有訊息
**原因**：JavaScript 可能沒有執行  
**解決方法**：
1. 確認 Console 篩選器為「All levels」
2. 檢查頁面是否有 404 錯誤（開發者工具 → Network）
3. 查看頁面原始碼，確認是否為最新版本
