# 手機數據問題修復說明

**修復時間**：2026-03-13 20:57

## 🔍 問題診斷

### 原因分析
**核心問題：** localStorage 在手機和電腦是分開的

**詳細說明：**
1. 電腦上使用系統 → 數據儲存在電腦的 localStorage
2. 手機第一次開啟 → 手機的 localStorage 是空的
3. 系統沒有自動初始化數據 → 顯示空白

**技術原因：**
- 原本的初始化邏輯檢查 `quoteSystemInitialized` 標記
- 但沒有檢查實際數據是否存在
- 如果數據被清除但標記還在，會導致不初始化

---

## ✅ 解決方案

### 修改 1：data.js 改進初始化邏輯

**之前的邏輯：**
```javascript
function initializeData() {
    // 只檢查標記
    if (localStorage.getItem('quoteSystemInitialized')) {
        return; // 有標記就不初始化
    }
    // ...初始化代碼
}
```

**改進後的邏輯：**
```javascript
function initializeData() {
    // 檢查實際數據是否存在
    const hasProjects = localStorage.getItem('projects');
    const hasPriceTable = localStorage.getItem('priceTable');
    const hasUsers = localStorage.getItem('users');
    
    // 如果所有核心數據都存在，跳過初始化
    if (hasProjects && hasPriceTable && hasUsers) {
        console.log('✅ 數據已存在，跳過初始化');
        return;
    }
    
    console.log('🔄 開始初始化數據...');
    // ...初始化代碼
}
```

**改進效果：**
- ✅ 不依賴標記，檢查實際數據
- ✅ 任一核心數據缺失就重新初始化
- ✅ 確保數據真實存在

### 修改 2：新增通用數據檢查模組

**新增檔案：** `ensure-data.js`

**功能說明：**
```javascript
// 🔧 確保數據已初始化
function ensureDataInitialized() {
    const projects = localStorage.getItem('projects');
    const priceTable = localStorage.getItem('priceTable');
    const users = localStorage.getItem('users');
    
    // 如果任一核心數據不存在，觸發初始化
    if (!projects || !priceTable || !users) {
        console.log('⚠️ 數據缺失，正在初始化...');
        
        if (typeof initializeData === 'function') {
            localStorage.removeItem('quoteSystemInitialized');
            initializeData();
            console.log('✅ 數據初始化完成');
            return true;
        } else {
            console.error('❌ initializeData 函數不存在！');
            alert('系統數據載入失敗，請重新整理頁面！');
            return false;
        }
    }
    return true; // 數據已存在
}

// 頁面載入時自動檢查
window.addEventListener('DOMContentLoaded', function() {
    ensureDataInitialized();
});
```

**特色：**
- ✅ 頁面載入時自動執行
- ✅ 檢測數據缺失並自動修復
- ✅ 提供清楚的錯誤提示
- ✅ 適用於所有頁面
